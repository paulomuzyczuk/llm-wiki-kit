# Role Map — Code Craftsperson

A curated entry point for the **code-craftsperson** role: someone writing, reviewing, or
improving code day to day. This is a *map of content* (a navigation page), not a synthesis
— it points at the pages most worth reading for this role, ordered by where you'd start.

> At this example vault's scale (a handful of pages) a role map is deliberately short.
> In a mature vault a role MOC curates the 10–15 highest-value pages for the role and is
> the recommended browse layer above the subject-grouped index (`wiki/index.md`).

## Start here

1. [[version-control]] — the daily substrate: commits, branches, pull requests, and the
   discipline (version-everything, singularity of information) that keeps history honest.
2. [[code-review]] — how to run conspicuous commit review as a standing practice, and why
   "more eyes on commits" is the cheapest quality lever you have.
3. [[continuous-integration]] — the mechanical counterpart to review, and a grounded read
   on what CI does and doesn't buy you (throughput over per-PR latency).
